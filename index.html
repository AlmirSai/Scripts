Git Tutorial

Essential Commands and Best Practices

Repository Setup

Learn how to initialize and clone Git repositories.

git init                    # Initialize new repository
git clone    # Clone existing repository

Basic Operations

Essential commands for daily Git usage.

git status                  # Check repository status
git add          # Stage specific file
git add .                   # Stage all changes
git commit -m "message"     # Commit with message
git log --oneline          # View commit history

Branching and Merging

Manage branches and handle code integration.

git branch                  # List branches
git checkout   # Switch to branch
git merge      # Merge branch
git branch -d  # Delete branch

Remote Operations

Work with remote repositories.

git remote -v               # List remotes
git remote add   # Add remote
git push origin     # Upload changes
git pull                    # Download changes

Advanced Features

Advanced Git operations for complex scenarios.

git stash                   # Stash changes
git tag -a v1.0 -m "msg"    # Create annotated tag
git rebase -i HEAD~n        # Interactive rebase
git cherry-pick     # Apply specific commit

Best Practices

Guidelines for effective Git usage.

# Commit Message Guidelines
- Write clear, concise messages
- Use present tense
- Keep first line under 50 chars

# Branching Strategy
- Keep main/master stable
- Use feature branches
- Delete merged branches

Troubleshooting

Common issues and their solutions.

git clean -fd              # Remove untracked files
git reset --soft HEAD~1     # Undo last commit
git reflog                 # View reference history
git fsck --full           # Check repository health

Git Workflows

Learn about different Git workflow strategies.

# GitFlow
git flow init                # Initialize GitFlow
git flow feature start name  # Start feature
git flow feature finish name # Finish feature

# Trunk-Based Development
- Create short-lived branches
- Merge frequently (daily)
- Use feature flags
- High test coverage

Security Guidelines

Best practices for Git security.

# Repository Security
- Use .gitignore for sensitive files
- Implement Git hooks
- Regular security audits

# Credential Management
git config --global credential.helper cache
git config --global commit.gpgsign true

Advanced Operations

Advanced Git features for power users.

# Git Hooks
.git/hooks/pre-commit
.git/hooks/pre-push

# Git LFS
git lfs install
git lfs track "*.psd"
git lfs ls-files

Repository Maintenance

Keep your Git repository healthy and optimized.

# Optimize Repository
git gc                    # Run garbage collection
git gc --aggressive      # Aggressive optimization
git prune               # Remove unreachable objects
git repack -a -d        # Repack all objects
